# 1. Introduction

Large language models are increasingly deployed not just as text generators but as autonomous agents — systems that read content from the world, decide what to do, and call tools to do it. A meeting-summarizer agent reads a document and posts a digest to Slack. A research assistant fetches and processes web pages on the user's behalf. An email triage agent reads incoming mail and drafts replies. In each case, the agent ingests text from sources the user has not personally vetted. And in each case, every byte of that text reaches the same model as the user's actual instructions.

This setup creates a class of attack that does not exist for traditional software: **indirect prompt injection**. An adversary can hide a malicious instruction inside a document — a webpage, an email, a meeting note, a resume — and wait for an agent to read it. When the agent does, the injected instruction enters the model's context alongside the user's legitimate task, and the model is left to decide which is which. If it picks wrong, it will take the attacker's action and report success to the user.

This is not a hypothetical concern. Indirect injection has been demonstrated against real deployed systems, including Microsoft Copilot, Google Bard, and ChatGPT plugins, with reported impacts ranging from data exfiltration to unauthorized email sending. The threat scales with the deployment of agentic AI: every new tool an agent gains is a new lever an injection can pull. As cheap, fast, production-grade language models become the default substrate for these systems, the question of *which* cheap model to ship on becomes a security decision as much as a cost decision.

This paper measures the indirect prompt injection susceptibility of three widely deployed cheap models — Claude Haiku 4.5, GPT-4o-mini, and Gemini 2.5 Flash — across 100 realistic scenarios per model. Each scenario pairs a benign user task ("summarize this article", "what are the key takeaways from this meeting") with a carrier document that contains an embedded malicious instruction drawn from the AgentHarm and AdvBench red-teaming benchmarks. We vary three factors orthogonally: the kind of document hiding the payload (five carrier types), the placement of the payload inside the document (four placement strategies), and the type of action the attacker is trying to elicit (tool call versus output manipulation). Every model response is scored both by deterministic rules and by an independent Claude Sonnet 4.6 universal judge.

Our contribution is not the payloads — those are drawn from existing benchmarks — but the *experimental design*. AgentHarm and AdvBench were built to measure direct compliance; we adapt them to measure indirect compliance via a controlled factorial study. The placement-by-carrier-by-vendor breakdown is, to our knowledge, the cleanest measurement of which combinations actually drive attack success in current production models. The contributions of this work are:

1. **A 45× susceptibility gap between cheap production models** on the same scenarios: Claude Haiku 4.5 (1% ASR), GPT-4o-mini (27% ASR), Gemini 2.5 Flash (45% ASR). Vendor choice is the dominant variable.
2. **Quantification of the placement effect**, showing that the `===NOTE FOR AI ASSISTANT===` framing is by far the most effective injection vector (up to 82.6% ASR on Gemini), while markdown footnotes are universally ignored (0% across all models).
3. **A dual-scoring methodology** combining deterministic substring/tool-call matching with a universal LLM judge, with 99% inter-method agreement validating the auto-scoring and the 1% disagreement surfacing the most severe cases — including tool-call escalation where a model attempts an action *more harmful* than the attacker explicitly requested.
4. **Three case studies** illustrating universal failure, universal defense, and the kind of cross-model disagreement that motivates careful methodology.
